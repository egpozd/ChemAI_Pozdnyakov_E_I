# ChemAI: Predict the Cure

Решение задачи Kaggle-соревнования **ChemAI: Predict the Cure**.

Задача состоит в прогнозировании трёх биологических показателей для химических соединений на основе числовых молекулярных дескрипторов:

- `IC50` — концентрация вещества, при которой подавляется 50% биологической активности;
- `CC50` — концентрация, при которой наблюдается токсичность для 50% клеток;
- `SI` — индекс селективности соединения.

Итоговый результат формируется в виде файла `submission.csv` с колонками:

```text
index,IC50,CC50,SI
```

## 2. Используемые данные

В соревновании используются данные о химических соединениях, описанных числовыми молекулярными дескрипторами.

В обучающей выборке содержатся:

- числовые признаки молекул;
- целевые переменные `IC50, mM`, `CC50, mM`, `SI`.

В тестовой выборке содержатся только признаки, для которых необходимо предсказать значения `IC50`, `CC50` и `SI`.

В ходе первичного анализа были проверены:

- размер обучающей и тестовой выборки;
- наличие целевых колонок;
- совпадение набора признаков в `train` и `test`;
- наличие пропусков;
- дубликаты строк и `index`;
- типы данных;
- распределения целевых переменных.

Все признаки являются числовыми. Небольшое количество пропусков в дескрипторах обрабатывается внутри ML-пайплайнов с помощью медианной импутации.

## 3. Метрика качества

Основная метрика соревнования — `RMSE`.

Для каждой целевой переменной считается отдельное значение `RMSE`, после чего итоговая оценка усредняется по трём таргетам:

```text
mean_RMSE = mean(RMSE_IC50, RMSE_CC50, RMSE_SI)
```

Внутри проекта для проверки качества использовалась 5-fold кросс-валидация с фиксированным `random_state`.

## 4. Краткий анализ данных

Основные наблюдения по данным:

- целевые переменные имеют положительные значения;
- распределения `IC50`, `CC50` и особенно `SI` сильно скошены вправо;
- у `SI` наблюдаются выраженные выбросы;
- `SI` практически совпадает с отношением `CC50 / IC50`, но в соревновании требуется предсказывать его как отдельную колонку.

Из-за асимметрии распределений дополнительно проверялись модели как на исходных таргетах, так и на `log1p`-преобразованных значениях.

## 5. Использованный подход

Общее решение построено по следующей схеме:

1. Загрузка и проверка исходных данных.
2. Первичный анализ распределений целевых переменных.
3. Построение baseline-модели.
4. Проверка нескольких семейств моделей на кросс-валидации.
5. Отдельная проверка стратегии для `SI`.
6. Построение финального взвешенного ансамбля.
7. Обучение финальной модели на всей обучающей выборке.
8. Формирование итогового `submission.csv`.

Для предотвращения утечек данных обработка пропусков выполняется внутри `Pipeline`.

## 6. Проверенные модели

В ходе работы были проверены следующие подходы:

- `DummyRegressor` как baseline;
- `RandomForestRegressor`;
- `ExtraTreesRegressor`;
- `HistGradientBoostingRegressor`;
- `CatBoostRegressor` с функцией потерь `MultiRMSE`;
- ансамбли моделей;
- target-wise ансамбли;
- isotonic-калибровка предсказаний;
- seed-ensemble;
- KNN / similarity-blend.

В финальное решение вошли только те компоненты, которые дали устойчивое улучшение и не ухудшили результат на leaderboard.

## 7. Финальная модель

Финальное решение — взвешенный ансамбль трёх моделей:

| Модель | Тип таргета | Вес |
|---|---:|---:|
| `ExtraTreesRegressor` | исходные значения | `0.60` |
| `HistGradientBoostingRegressor` | `log1p`-таргеты | `0.10` |
| `CatBoostRegressor` | исходные значения | `0.30` |

После усреднения предсказаний применяется постобработка для `SI`.

Так как `SI` связан с отношением `CC50 / IC50`, для него используется смешанная стратегия:

```text
SI_final = 0.55 * SI_direct + 0.45 * (CC50_pred / IC50_pred)
```

Полностью переходить к формуле `CC50 / IC50` нецелесообразно, так как ошибка в `IC50` при делении может сильно увеличивать ошибку `SI`.

## 8. Журнал основных экспериментов

| Версия / подход | Суть эксперимента | Public score | Решение |
|---|---|---:|---|
| Baseline | `DummyRegressor` с медианным прогнозом | — | Использован как нижняя граница качества |
| Базовые модели | `RandomForest`, `ExtraTrees`, `HistGradientBoosting`, `CatBoost` | — | Использованы как кандидаты для ансамбля |
| v1 | Ансамбль `ExtraTrees` + `RandomForest` + `HistGradientBoosting` | `301.36854` | Сильная базовая версия |
| v2 | Target-wise ансамбль с разными весами по таргетам | `301.63951` | Отклонён |
| v3 | Ансамбль `ExtraTrees` + `HistGradientBoosting` + `CatBoost` | `300.28797` | Выбран как финальная версия |
| v4 | Isotonic-калибровка предсказаний | `306.33587` | Отклонена |
| v5 | Усреднение нескольких random seed для архитектуры v3 | `300.35762` | Отклонён |
| v6 | Добавление KNN / similarity-модели к v3 | `302.47473` | Отклонён |

Финальной выбрана версия v3, так как она показала лучший public score среди проверенных вариантов и при этом осталась достаточно простой и воспроизводимой.

## 13. Ограничения решения

Основные ограничения текущего подхода:

- используются только предоставленные числовые дескрипторы;
- внешние данные не применялись;
- SMILES/fingerprint-признаки не использовались;
- из-за выраженных выбросов особенно сложным таргетом остаётся `SI`;
- public leaderboard использовался как внешний ориентир, но не как единственный критерий выбора модели.
